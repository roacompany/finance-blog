## 📊 토스 블로그 분석 결과

### 분석 대상
- **토스피드 (blog.toss.im)**: 금융 라이프스타일 콘텐츠
- **toss.tech**: 기술 블로그 (참고용)
- **토스 UX 라이팅 가이드**: 공식 원칙

### 핵심 발견사항

#### 1. 제목 스타일
```
❌ 격식: "예적금 금리 비교 분석"
✅ 토스: "왜 금리가 같아도 적금 이자보다 예금 이자가 많을까?"
✅ 토스: "주택연금, 주택담보대출과 뭐가 다를까?"
```

**패턴:**
- 질문형 (90% 이상)
- "왜/어떻게/무엇이" 시작
- 일상 언어 사용
- 호기심 유발

#### 2. 톤앤매너 원칙
**토스의 5가지 핵심 원칙:**
1. **Clear** (명확한): 한 번에 이해 가능
2. **Concise** (간결한): 꼭 필요한 내용만
3. **Casual** (친근한): 해요체, 말 거는 느낌
4. **Respectful** (존중하는): 사용자 시간 존중
5. **Empathetic** (공감하는): 감정 이해

#### 3. 콘텐츠 구조 패턴

**A. 라이프스타일형** (월간 토스픽, 에브리데이 경제학)
```
제목: [일상 상황] + [금융 개념]
예: "조선시대에는 사교육비로 얼마를 썼을까?"
예: "관중 1200만 시대, 한국 프로야구 인기의 비결"

구조:
1. 흥미로운 도입부 (일화, 사건)
2. 연결고리 (→ 금융으로 자연스럽게)
3. 핵심 정보 (간단명료)
4. 생활 팁 (실천 가능한)
```

**B. 금융 설명형** (토스피드 메인)
```
제목: "왜 [현상]일까?" or "[용어], [용어2]와 뭐가 다를까?"

구조:
1. 궁금증 유발 질문
2. 즉시 용어 설명 (그때그때)
3. 비교 표 (시각화)
4. 실생활 예시
5. 요약 (3줄)
```

**C. 혜택/가이드형**
```
제목: "[서비스명] [혜택/사용법] 총정리 2026년 [월]"

구조:
1. 새로 나온 것 (NEW)
2. 단계별 설명 (1,2,3)
3. 주의사항 (강조)
4. FAQ
```

#### 4. 용어 설명 방식

**즉시성 원칙:**
```
❌ 나쁜 예:
"COFIX는 Cost of Funds Index의 약자입니다."

✅ 토스 방식:
"COFIX는 은행이 돈을 모으는 데 드는 평균 비용이에요.
(Cost of Funds Index의 약자)"
```

**그때그때 풀기:**
```
"주택연금은 집을 담보로 맡기고 매달 돈을 받는 거예요.
(= 역모기지론)

주택담보대출은 집을 담보로 목돈을 빌리는 거예요.
(= 일반 모기지론)"
```

## 🎨 토스 스타일 작성 원칙

### 제목 작성법

**DO ✅**
```
"왜 예금보다 적금 이자가 적을까?"
"월세 vs 전세, 나한테 맞는 건?"
"2026년 1월 달라진 금융 혜택 총정리"
```

**DON'T ❌**
```
"예금과 적금의 금리 구조 비교 분석"
"월세 및 전세의 장단점에 대한 고찰"
"2026년 금융 제도 변경사항 안내"
```

### 톤앤매너 변환표

| 기존 (격식체) | 토스 스타일 (친근체) |
|--------------|---------------------|
| ~입니다 | ~예요/이에요 |
| ~하시기 바랍니다 | ~해보세요/하시면 돼요 |
| 귀하 | (삭제) or 여러분 |
| ~에 대해 설명드리겠습니다 | ~을 알려드릴게요 |
| 상기와 같이 | 위에서 본 것처럼 |
| ~할 수 있습니다 | ~할 수 있어요 |

### 문장 길이 원칙

- **한 문장**: 최대 1.5줄 (모바일 기준)
- **한 문단**: 2-3문장
- **핵심 요약**: 무조건 3줄

### 용어 설명 타이밍

```
❌ 나쁜 예:
본문에서 COFIX를 10번 사용한 뒤,
하단에 용어 설명

✅ 토스 방식:
COFIX를 첫 등장 시점에 바로 설명
(괄호나 Callout 사용)
```
